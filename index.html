Prompting
Generic tutor prompt fails Kenyan learners
EduSavvy uses one undifferentiated prompt — "Explain this topic to a student" — with no CBC curriculum alignment, no Swahili support, and no rural learner context.
Students in Makueni and Turkana receive explanations using UK/US examples irrelevant to their lives, causing comprehension failure and dropout.
Framework → AIM + MAP
Fluency
Temperature 0.9 on exam revision generates hallucinations
The platform uses default high temperature for all interactions — including KCSE exam revision quizzes — producing inconsistent and sometimes fabricated answers.
Form 4 students memorise incorrect facts generated by a hallucinating AI, leading to exam failure and lost university entry.
Framework → 4D Framework
Ethics
Child learning data shared with advertisers without consent
Granular student data (quiz scores, failure points, learning pace) is aggregated and shared with edtech advertisers. No explicit consent mechanism exists for minors.
Children's learning vulnerabilities are monetised without parental knowledge, violating Kenya's Data Protection Act 2019 Sections 25 and 30.
Framework → TRACK + OASIS
Agents
Single agent teaches and grades the same student
One AI agent handles both content delivery and summative grading with no role separation, no authority limits, no memory layering, and no escalation to human teachers.
Students with learning disabilities receive punitive AI grading from the agent that teaches them, with no human override possible.
Framework → RANK + TRAIL

Prompting AIM + MAP — CBC-aligned tutor prompt

Rewritten prompt with explicit AIM and MAP labels:

A — Action You are an expert tutor for Kenyan secondary school students following the CBC curriculum. I — Intent Your goal is to explain [TOPIC] so the student builds genuine understanding — not memorisation. Use local Kenyan examples: markets, matatus, M-Pesa, maize farming, the Rift Valley. M — Mission Every explanation must be accessible to a student studying in English and Swahili, at Form [LEVEL], on a 2G connection. M — Mode Conversational tutor: patient, encouraging, never condescending. A — Audience Kenyan Form 3 or 4 student, age 15–18, rural or peri-urban, CBC curriculum. P — Purpose Prepare the student to answer KCSE exam questions confidently while understanding the real-world relevance of the concept.
Fluency 4D Framework — KCSE exam revision interaction
Discover Student asks: "Explain photosynthesis for my KCSE revision." Define Context: KCSE Biology, Form 3, Chapter 4, CBC syllabus. RAG Source: KNEC KCSE Biology Syllabus 2024 + KIE-approved textbook corpus. Temperature: 0.2 ← low for exam factual accuracy. Negative prompt: Do NOT use non-Kenyan ecosystems. Do NOT generate content outside the KNEC syllabus. Do NOT speculate. Design Response structure: 1. Definition 2. Light reaction 3. Dark reaction 4. Real-world application (maize farming, Rift Valley) 5. Bilingual English/Swahili glossary appended. Deliver Plain text, low-bandwidth mode. Optional Swahili audio narration toggle.
Ethics TRACK + OASIS — student data governance
T — Transparency Full disclosure in Swahili + English at onboarding. No hidden telemetry. R — Respect Parental consent required for all learners under 18. Granular: learning data | platform improvement | research — three separate toggles. A — Accountability Named DPO registered under Kenya DPA 2019. C — Control Parents can download, amend, or delete child's data at any time. K — Knowledge Annual plain-language privacy report published for all users. O — Oversight DPO reviews all 3rd-party sharing agreements quarterly. A — Auditing Automated audit logs retained 3 years (DPA 2019 requirement). S — Safety Student data NEVER sold. Used only for adaptive learning improvement. I — Inclusion Privacy UX in Kikuyu, Luo, Kalenjin — visual icons for low-literacy parents. S — Sustainability Data minimisation: collect only what the learning outcome strictly needs. Regulation: Kenya Data Protection Act 2019, §25 (lawful basis), §30 (children's data).
Agents RANK + TRAIL — 2-agent handoff architecture

🤖 TutorBot — Tier 1

  • Delivers CBC curriculum content
  • Generates practice questions
  • Session memory only
  • Cannot grade summative work
  • Cannot store beyond session
→

📋 AssessBot — Tier 2

  • Grades vs KNEC rubrics
  • Flags learning gaps
  • Stores term-level memory
  • Cannot modify content
  • Cannot override teacher
→

👩‍🏫 Human Teacher — Tier 3

  • Full override authority
  • Reviews flagged decisions
  • Accesses persistent memory
  • Can suspend any agent
TRAIL log fields: timestamp | student_id (anonymised) | agent | action | confidence | handoff_reason Memory layers: Session → TutorBot only (cleared post-session) Term → AssessBot only (retained per academic term) Persistent → Human Teacher (never exposed to agents) Handoff triggers: → AssessBot confidence < 0.75 → escalate to teacher within 2 min → Student distress keyword detected → both agents pause immediately
🔴
Kill switch: any teacher or parent suspends both agents instantly via one dashboard button. All sessions terminate within 5 seconds. No restart without human re-authorisation. Every activation triggers an automatic incident report to the DPO.

Section 4 · Safari Reflection
How this course changed my thinking

Across all four terrains

Savannah · Tsavo · Ethical Savannah · Agentic Pride

Before this course, I thought AI strategy meant choosing the right model and writing a good prompt. The AI Safari taught me that strategy — especially in an African context — is fundamentally about power: who holds it, who it affects, and who it forgets.

The moment my thinking shifted
It happened in the Ethics Savannah terrain, when I applied TRACK to EduSavvy's data practices. I had assumed that collecting student data to improve learning was inherently good. But when I mapped it against Kenya's Data Protection Act 2019 and asked: "who consented to this, and in what language?" — everything changed. A parent in Turkana County who cannot read English has never truly consented, even if a checkbox was ticked. A system can be technically functional and ethically hollow at the same time.

The HORIZON scan changed how I frame impact. I used to measure edtech success as "students who pass exams." Now I see three rings: the learner, the family and teacher around them, and the physical environment the infrastructure touches. AI that ignores those outer rings is not intelligent — it is incomplete.

RANK + TRAIL changed how I think about AI autonomy. I used to see agents as efficient replacements for slow human processes. Now I see them as systems that must earn authority incrementally — every action logged, every handoff justified, and a human always able to pull the plug.

Africa does not need AI imported from elsewhere and relabelled. It needs AI designed from the inside — with local language, local regulation, local consequence, and local ambition at the centre. This course gave me the frameworks to build that, and the honesty to ask whether I am actually doing it.
